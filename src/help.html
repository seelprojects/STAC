Project Settings

This section gives a brief description about project settings.

Selecting a project to index

  • Click on Open Project button or press (ALT + O).
  • In Browse For Folder dialog, select a project (Java, C# or C++) and click OK.
  • Alternatively, a full path for the project folder can also be typed in the textbox followed Enter key.

Selecting files to index

  • Select a project to index.
  • Available Files: Available Files are files in the project that are excluded from indexing. Initially all project's files are in this list. User can select all Available Files for indexing by clicking >> button or select individual file by clicking > button.
  • Selected Files: Selected Files are files in the project that are used for indexing. User can remove all Selected Files for indexing by clicking << button or remove individual Selected File by using < button.

Indexer Settings

This section gives a brief description about indexer settings.

Selecting splitting technique

STAC splits by default all words with camel-casing. Advance splitting can be used in indexing process to split words that are not properly camel-cased. Check Advance Splitting checkbox to apply advance splitting. Uncheck the box to apply only camel-case splitting.

Spell Checking

Incorrect spellings can be checked by checking Spell check checkbox. A list of misspelled words and their corrections is displayed in Spell checking tab once the project is indexed.

Stemming

STAC uses Porter Stemmer (http://www.tartarus.org/~martin/PorterStemmer) to reduce word to its morphological root by removing derivational and inflectional suffixes. Porter stemmer can be used during indexing process by checking Stemmer checkbox. A list of words and their root is displayed in Stemming tab once the project is indexed.

Lemmatization

STAC provides Stanford coreNLP (http://stanfordnlp.github.io/CoreNLP/) to reduce a word to its lemma. Lemmatization can be used during indexing process by checking Lemmatizer checkbox. A list of words and their root is displayed in Stemming tab once the project is indexed.

Custom Dictionary

STAC's Custom Dictionary allows users to maintain their own list of words that are neither splitted nor spell checked during indexing. This feature is accessed from File menu.

  • Click on File menu.
  • Click Custom Dictionary.
  • To add a single word: Type the word in the text box and click Add Word. Press Save to save the changes.
  • To delete a single word: Select the word from the list and click Delete Selected. Press Save to save the changes.
  • To delete all words: Click Delete All. Press Save to save the changes.
  • To import from other text file: User can import list of words from other text file. Click Import and select the file to be used. Then press Open. Press Save to save the changes.

Saving Result

Result from an indexing process can be saved using STAC's Save menu (File menu > Save)

  • Click on File menu.
  • Click Save.
  • Select a folder where the result text files will be saved and press OK.

Project Metrics

Project metric displays number of source files and lines of codes and comments. These numbers are available after a project is selected to indexed.

Setting up the Stanford Lemmatizer

  • Download Stanford Core-NLP (v3.5.2) from http://stanfordnlp.github.io/CoreNLP/ and extract stanford-corenlp-3.5.2-models.jar. Extract the content of the jar file to a folder, let's say x.
  • Open the folder where STAC is installed and locate SourceCodeIndexer.UI.exe.config
  • Open the file using any text editor. Update the value for StanfordModelsPath with the folder's path where stanford models were extracted (here it's path to x folder).
  • Close and re-run STAC.
